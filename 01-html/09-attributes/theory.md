# HTML Attributes: id, class, data

## Обобщение
- Атрибутът е параметър на HTML елемент, който добавя допълнителна информация или поведение.
- Пише се в отварящия таг, например: `<a href="https://example.com" target="_blank">Линк</a>`.

## ID атрибут
- Уникален идентификатор на елемент
- `<div id="unique-id">Съдържание</div>`
- Използва се с CSS и JavaScript
- Само един за елемент!

## Class атрибут
- Множествен селектор за групи елементи
- `<div class="card highlight">Съдържание</div>`
- Един елемент може да има множество класове (разделени със пространство)

## Data атрибути
- Съхранява допълнителна информация
- `<div data-user-id="123" data-role="admin">Потребител</div>`
- Достъпни от JavaScript

## Кога да ползваш?
- ID: уникални елементи (nav, footer)
- Class: повторяемия стаил (cards, buttons)
- Data: допълнителна информация за JavaScript

## Честа грешка
- Използване на един class за много неродствени елементи
- Дублиране на id-та

## Бележки от урока

### Главни атрибути (основа)

- id: уникално име за един конкретен елемент на страницата.
- class: име на група за повтарящи се елементи и стилове.
- data-*: допълнителни данни за логика, най-често за JavaScript.

### Допълнителни атрибути (често използвани)

1. Общи (global)
- title: подсказка при задържане с мишка.
- lang: език на съдържанието.
- dir: посока на текста (ltr/rtl).
- hidden: скрива елемента.
- tabindex: ред на фокус при работа с клавиатура.

2. За линкове (a)
- href: адресът на линка.
- target: къде да се отвори (например нов таб).
- rel: допълнителна безопасност и поведение при target.
- download: позволява сваляне на ресурс.

3. За изображения (img)
- src: път до изображението.
- alt: текстово описание за достъпност.
- width/height: размери.
- loading: контрол на зареждането (например lazy).

4. За форми
- name и value: име и стойност на поле.
- placeholder: подсказка в полето.
- required: задължително поле.
- minlength/maxlength: контрол на дължината.
- pattern: шаблон за валидиране.
- checked/selected: начално състояние на избор.
- disabled/readonly: ограничаване на редакция.

5. За медия
- controls: показва контролите за пускане.
- autoplay: автоматично стартиране.
- loop: повторение.
- muted: без звук.
- poster: начално изображение за video.

6. За достъпност
- aria-label: кратко описание за screen readers.
- aria-describedby: връзка към по-подробно описание.
- aria-hidden: скриване от assistive технологии при нужда.

Практично правило:

- Използвай първо семантичен елемент.
- Добавяй само атрибути, които носят конкретен смисъл.
- Избягвай излишни атрибути без реална полза.
